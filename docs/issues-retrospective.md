# MICS009 問題点レトロスペクティブ（〜v0.3.40）

screen-spec.md §0-54 の記録を「再発防止に効く分類」で整理したもの。詳細な個別記録は screen-spec.md を参照。

## A. 音・エンジン系（楽器としての致命傷）

| 問題 | 原因 | 対策 | 版 |
|---|---|---|---|
| Mic録音すると全体音量が落ちる | `getUserMedia({audio:true})`＝AECデフォルトONで、ブラウザが「通話中」とみなし出力をダッキング | echoCancellation/autoGainControl/noiseSuppression を全OFF | v0.3.37 |
| 録音が表示されるのに鳴らない | 同上（AECがマイクをゲート→ほぼ無音で録れる） | AEC OFF＋無音ガード | v0.3.38 |
| 演奏（キー発音）が重い | selectPadの重いUI同期が発音経路に同居 | rAF集約で発音とUI更新を分離 | v0.3.21 |
| レイテンシ | SSM2044 Workletを全グループに通していた | 上物(g0/g1)はバイパス | v0.3.23 |

**教訓**: 発音経路に「音以外の仕事」を混ぜない。ブラウザの"通話向け"デフォルトはサンプラーには全部害。

## B. iPhone/モバイル実機系（PCプレビューで見えない崩れ）※最多カテゴリ

| 問題 | 原因 | 版 |
|---|---|---|
| ロゴがステータスバー時刻に重なる | `viewport-fit=cover`なのにセーフエリア余白ゼロ | v0.3.36 |
| 画面がスクロールしてしまう | `body{overflow:hidden}`だけで`html`が動く（iOS Safariの仕様） | v0.3.36（iPad同種 v0.3.17） |
| strip-head（LOAD/SMPL行）が全幅にならない | デスクトップ用`align-items:start`がモバイルに残留 | v0.3.36 |
| パッド右に空きができる | split用CSSの特異度競合 | v0.3.26 |
| MENUモーダルの崩れ | 設定行の折返し | v0.3.33 |
| CHOP行が2行に折返す／Pitch・Levelが幅一杯でない | flex-wrap＋`max-width:140px`残留 | v0.3.39 |
| 録音オーバーレイがEDITモーダルに重なって見切れる | `top:84px`固定同士の衝突 | v0.3.40 |

**教訓**: 「デスクトップ用レイアウト規則（split/start/max-width/top固定）がモバイルに漏れる」が繰り返しパターン。セーフエリア・html scroll・実幅はエミュレーションで再現しないものがある。モバイル用リセットの一箇所集約が構造対策（未着手）。

## C. 状態の二重管理・UI迷子系（設計レベル）

| 問題 | 対策 | 版 |
|---|---|---|
| 同じ状態（音量/mute）が3画面から編集できて同期バグ | MIX画面を廃止＝Single Source of Truth化 | §8 |
| 8×2と4×4で再生ボタン位置が違う・用途が説明されない | 8×2レイアウト自体を撤去して4×4一本化 | v0.2.30 |
| PADS⇔SEQ切替で縦幅/タブがジャンプ | サイズ共通化 | v0.3.12/15 |
| STEPモードとP-LOCKの操作系が競合 | STEP廃止→Digitaktモデルに確定 | v0.3.11 |
| COPY/DUPの重複 | COPY汎用化＋「元→先で自動オフ」 | v0.3.29/34 |

**教訓**: 「同じものを2箇所から操作できる」は必ず同期バグになる。機能を直すより片方を消す判断（MIX廃止・8×2廃止・STEP廃止）が効いた。aki ruleとも一致。

## D. 導線・操作感（aki rule違反）

- サンプル追加の入口がperfから届かない → EDITモーダルに集約（v0.3.35）
- TEMPO/SWING/情報窓など上段の混雑 → メニュー送り＋1行化（v0.3.6-8）
- 極小フォント8px＋ズーム禁止（I6）— **未解決の継続課題**

## 総括：3つの繰り返しパターン

1. **ブラウザのデフォルトは楽器の敵**（AEC・autoplay・html scroll・セーフエリア）
2. **デスクトップCSSのモバイル漏れ**が最頻出。「特異度で上書き」の積み重ねになっており、モバイル用リセットを一箇所に集約する余地あり
3. **UIの重複＝バグの温床**。「消す」判断が一番効果的だった
